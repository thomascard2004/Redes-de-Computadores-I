Tipos de Códigos de Acesso

  • CAC (Channel Access Code): Identifica a piconet.
  • DAC (Device Access Code): Usado durante o pedido e resposta de conexão (paging procedure).
  • IAC (Inquiry Access Code): Utilizado no processo de inquérito, para descobrir dispositivos próximos.

Cabeçalho

O cabeçalho é composto por diversos campos de controle, e os pacotes podem ser de diferentes tipos, como: ID, NULL, POLL, FHS, DM1, DH1, AUX1, DM3, DH3, DM5, DH5, HV1, HV2, HV3, DV.

Dados

Os dados incluem a informação útil a ser transmitida, além de um campo de verificação de erro (CRC) para garantir a integridade da informação.

Estados Operacionais

O Bluetooth opera em quatro estados principais, com diferentes níveis de consumo energético:

  • Desconectado: Nenhuma comunicação ocorre.
  • Conectando (Page / Inquiry): Estado de descoberta ou tentativa de conexão.
  • Ativo: Comunicação de dados em andamento.
  • Baixo consumo:
    • Hold: Dispositivo conectado, mas sem atividade temporária.
    • Sniff: Dispositivo escuta com menos frequência para economizar energia.
    • Park: Conectado, mas sem transmitir/receber — pronto para voltar à ativa.
